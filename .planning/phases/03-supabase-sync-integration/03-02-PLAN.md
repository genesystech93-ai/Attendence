# Plan 03-02: Swap client-side mock database layers with real Supabase queries

This plan describes how we replace mock local data persistence with dynamic backend database syncing using the `@supabase/supabase-js` client SDK on both desktop and mobile clients.

---

## Proposed Changes

### [Client SDK Setup]
- Add `@supabase/supabase-js` to `desktop/package.json` and `mobile/package.json` dependencies if not already present.
- Create connection client instances utilizing environment configurations.

### [Desktop Services]
#### [MODIFY] [dataService.ts](file:///c:/Users/Suraj/Desktop/Attendence/desktop/src/services/dataService.ts)
- Replace mock localStorage writes inside `dataService` with live Supabase client database selections and mutations:
  - `login()` -> Authenticate with Supabase Auth or fetch profile matching email.
  - `checkIn()` -> Insert a new record to `attendance_logs` table.
  - `checkOut()` -> Update today's `attendance_logs` row with `check_out` timestamp.
  - `getAttendanceLogs()` -> Query `attendance_logs` filtered by user session.
  - `submitLeaveRequest()` -> Insert a new record to `leave_requests` table.
  - `getLeaveRequests()` -> Query `leave_requests` table.
